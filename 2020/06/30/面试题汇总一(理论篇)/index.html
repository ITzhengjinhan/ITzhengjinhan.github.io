1、从一个url地址到最终页面渲染完成，发生了什么？

本人总结了六步：

  1. DNS解析:将域名解析为IP地址
  2. TCP请求连接，即三次握手
  3. 浏览器发送请求
  4. 服务器响应请求
  5. 渲染页面（主要分为①创建DOM树、CSSOM树，②创建渲染树，③计算页面布局，④填充颜色，渲染完成）
  6. TCP断开连接，即四次挥手

2、TCP和UDP的区别

  1. TCP是面向连接的，UDP是无连接的
  2. TCP是可靠性传输，UDP是不可靠的

3、js闭包的理解？举例并说明其主要的作用！

  1. 闭包的定义：闭包是指有权访问另一个函数作用域中的变量的函数
  2. 产生条件：①.函数嵌套②.内部函数调用外部函数
  3. 常见应用：①.延长外部函数局部变量的生命周期
  4. 闭包的缺点：容易造成内存泄漏
    从外部读取函数内部的变量
function f1(){
     var n = 123;
     function f2(){    //f2是一个闭包
         alert(n)
     }    
     return f2;
 }
闭包：f2可以读取f1中的变量，只要把f2作为返回值，就可以在f1外读取f1内部变量。
子对象会一级一级向上寻找所有父对象的变量，反之不行。
js中函数内部可以读取全局变量，函数外部不能读取函数内部的局部变量。

5、js中this的指向问题，以及call/apply/bind的区别？

1.以函数的形式调用，this指向window。如fun(),相当于window.fun();
2.以方法的形式调用，this指向调用方法的那个对象；如stu.fun(),this指向stu;
3.以构造函数的形式调用，this是新创建的那个对象；

call和apply改变this指向，第一个参数都是指向的对象，第二个参数就不一样了，apply需要数组做参数，call需要全部展开逐个放进去，
然后对比下bind，与call类似，但其返回的是一个函数且不会立即执行，需要主动调用才会执行。

var name="小王",age=17;
var obj={
    name:"小张",
    objAge:this.age,
    fun:function(){
    console.log(this.name+"年龄"+this.age);
    }
}
obj.objAge;  // 17,this指向obj
obj.fun()  // 小张年龄 undefined

var name="盲僧";
function show(){
console.log(this.name)
}
shows()  // 盲僧,this指向window

10、跨域问题

跨域大家都知道，不同地址，不同端口，不同级别，不同协议都会构成跨域。例如：about.haorooms.com和www.haorooms.com都会构成跨域。总结起来只要协议、域名、端口有任何一个不同，都被当作是不同的域。
解决方法：CORS、jsonp和webSocket解决跨域问题，vue中可以通过配置proxy等

11、arguments类数组的含义？

arguments 是一个参数数组，是一个类数组对象，只在函数中使用
例如：

function printArgs() {
    console.log(arguments);
}
printArgs("A", "a", 0, { foo: "Hello, arguments" })；
执行结果是：
["A", "a", 0, Object]

扩展:1、ES6中的arguments

function func() {
    console.log(...arguments);
}
func(1, 2, 3);
执行结果是：
1 2 3
简洁地讲，扩展操作符可以将 arguments 展开成独立的参数。

2、arguments 转数组
Array.from() 是个非常推荐的方法，其可以将所有类数组对象转换成数组。

12、js中值类型和引用类型的区别

JavaScript中的变量类型有哪些？
（1）值类型（基本类型）：字符串（string）、数值（number）、布尔值（boolean）、undefined、null （这5种基本数据类型是按值访问的，因为可以操作保存在变量中的实际的值）(ECMAScript 2016新增了一种基本数据类型：symbol http://es6.ruanyifeng.com/#docs/symbol )
（2）引用类型：对象（Object）、数组（Array）、函数（Function）

// 值类型：number、string、boolean、undefined、null
var a = 100
var b = a
a = 200
console.log(b) // 100 保存与复制的是值本身

// 引用类型：对象、数组、函数
// 可以扩展属性
var a = {age:20}
var b = a
b.age = 21 
console.log(a.age) // 21

13、原型与原型链的理解

1.原型：
在JavaScript中，原型也是一个对象，原型的作用，则是实现对象的继承。
在js的所有函数对象中，都存在一个属性prototype，该属性对应当前对象的原型。
而所有的JavaScript对象，都存在一个_proto_属性，_proto_属性指向实例对象的构造函数的原型，理解起来就如下：

var p = new Person(); 
console.log(p._proto_=== Person.prototype)//true
p是实例对象，Person是p的构造函数，可以看出来p的_proto_属性指向构造函数Person的原型。

面试问题：用代码来解释一下js是如何通过原型进行继承的？

var parent = function(name){
 this.name = name;
}
parent.prototype.getName = function(){
 return this.name;
}
var son = new parent("huahua");
console.log(son.getName());//'huahua'
son继承了parent的原型中的函数属性getName。

2.原型链:排除Object的prototype的原型是null以外，所有的对象和原型都有自己的原型，对象的原型指向原型对象。
在层级多的关系中，多个原型层层相连则构成了原型链。
在查找一个对象的属性时，倘若在当前对象找不到该属性，则会沿着原型链一直往上查找，知道找到为止，如果到了原型链顶端，还没找到，则返回undefined。

3.constructor:constructor是构造函数创建的实例的属性，该属性的作用是指向创建当前对象的构造函数。
例如，son.constructor == parent;//true

14、Cookie、sessionStorage、localStorage的区别

共同点：都是保存在浏览器端的容器

  1. cookie在浏览器和服务器之间来回传递。而sessionStorage和localStorage不仅自动把数据发给服务器，且在本地保存
  2. cookie只有4KB大小，而sessionStorage和localStorage有5MB大小
  3. cookie常用来保存用户的登录状态，例如将用户id存储于一个cookie内。其只在设置的cookie过期时间之前一直有效
    sessionStorage仅在当前的浏览器窗口关闭前有效，localStorage永久有效，除非用户手动清除缓存等。

15、cookie和session的区别

  1. cookie数据存放在浏览器上，session存放在服务器上
  2. cookie不是很安全，别人可以分析存放在本地的cookie并进行cookie欺骗，而session相对安全些

17、前端优化问题

  1. 降低请求量（减少HTTP请求，lazyLoad、api请求等）
  2. 加快请求速度（DNS预解析，vue中路由懒加载，CDN分发等）
  3. 渲染速度（CSS放头部，js放尾部）

18、get与post请求的区别

  1. get产生一个TCP数据包，post产生两个TCP数据包
  2. 对于get请求，浏览器会把http header 和data一并发送过去，服务器响应200返回数据
    而对于post，浏览器先发header，服务器响应100，浏览器再发送data，服务器响应200返回数据
    进行说明post安全性比get高

  2. position:absolute(left:50%;top:50%;transform:translate(-50%,-50%))

20、清除浮动的方法

  1. 在浮动元素后使用一个空属性（clear：both）
  2. 使用CSS给浮动元素添加overflow属性（overflow：hidden或overflow：auto）

21、position相关属性/css定位

  1. fixed（固定定位）：相对浏览器窗口固定定位，不随滑轮滚动，脱离文档流
  2. relative（相对定位）：相对原来位置定位，未脱离文档流
  3. absolute（绝对定位）：相对最近的父元素来定位，脱离文档流，通常使用“子绝父相对”的方式定位，使得子元素相对父元素定位

22、CSS选择器

div,p    选择所有<div>元素和<p>元素
div p    选择<div>元素内的所有<p>元素
div>p    选择所有父级是 <div> 元素的 <p> 元素
div+p    选择所有紧接着<div>元素之后的<p>元素（只有一个）
div~p    选择div元素同级的每一个p元素
div:hover p    选择鼠标悬浮在上面时选择其子级p标签
div:hover~p    选择鼠标悬浮在上面时选择其同级p标签
div:before    在<div>元素之前插入content内容
div:after    在<div>元素之后插入content内容
