uni-app在开发中遇到的问题

无标签
发布日期:   2020-10-28

一、uni.navigateBack传递参数到上一个页面

方法一：采用$emit，$on以及$off
1.点击返回的这个页面触发一个事件，使用uni.$emit("事件名",对象参数)
    confirm(){
        uni.$emit('CHEvent',{CHData:this.CHData})
        uni.navigateBack({
            delta:1
        })
    },
2.在上一个页面的onShow或者onload中添加监听一个事件
    onLoad() {
        uni.$off("CHEvent");//注意：绑定监听前应解除绑定，避免产生多个监听事件
        uni.$on('CHEvent',function(data){
            console.log(data)
            })
        },
方法二：直接修改上个页面的data里的数据
1. B页面
    let pages = getCurrentPages();  //获取所有页面栈实例列表
    let nowPage = pages[ pages.length - 1];  //当前页页面实例
    let prevPage = pages[ pages.length - 2 ];  //上一页页面实例
    prevPage.$vm.navCity = "全世界";   //修改上一页data里面的searchVal参数值为1211
    uni.navigateBack({  //uni.navigateTo跳转的返回，默认1为返回上一级
        delta: 1
    });
2. A页面
  data() {
        return {
            navCity: "全国"
        }
    }

二、uniapp的rich-text标签无法识别一些旧标签问题

一些旧标签无法识别

<strong>=<b> :定义重要性强调的文字
<em>=<i> ：定义斜体
<ins>=<u>：定义下划线
<del>=<s>：定义删除的文字
因为<strong>,<em>,<ins>,<del>这些标签有语意，所以uniapp中取代了<b>,<u>,<i>,<s>。
但是在富文本编辑器中往往出现旧标签，从而导致无法识别。

解决方案：
不能识别就替换：
var str = '<u>人工智能</u>是被机器展示的智力年10月 ，美国白宫发布了《为未来<u>人工智能</u>做好准备'
    str=str.replace("<u>/g","<ins>")
    str=str.replace("<\/u/g","<\/ins>")

扩展：去掉标签里的内容

string sXML = "ABCDEFCSF<p>123123</p>KOPL";//现在我要去掉P标签中的内容，实现ABCDEFCSFKOPL这样的效果。
正则方案：
Regex rg = new Regex("<pict>.*</pict>");
　　sXML = rg.Replace(sXML, "");

三、uniapp获取当前页面路由及参数

let routes = getCurrentPages(); // 获取当前打开过的页面路由数组
let curRoute = routes[routes.length - 1].route //获取当前页面路由
let curParam = routes[routes.length - 1].options; //获取路由参数
// 拼接参数
let param = ''
for (let key in curParam) {
param += '&' + key + '=' + curParam[key]
}

四、v-model使用模板字符串拼接变量会报错

v-model="`字符串${index}`"会报错
解决方案：v-model="radioSelect['radio'+index]"

radioSelect:{
    radio0:'',
    radio1:'',
    radio2:''
},

五、改变数据域data的变量，没有双向绑定到view层渲染

this.$forceUpdate();//强制刷新

六、vue如何获取dom对象的信息

ref方法

  1. 首先有一个元素
    <div class="parent">
<div class="son" ref="btn">
</div>
</div>
  2. 通过this.$refs获取son的dom
    this.$refs.btn.$el.style.position = ‘fixed’
  3. 获取son的父元素
    this.$refs.btn.$el.parentNode.style.position = ‘fixed’
  4. 获取dom的属性
    this.$refs.btn.$el.getBoundingClientRect()
    更多文档参考：
    https://developer.mozilla.org/zh-CN/docs/Web/API/Element/getBoundingClientRect

七、uni-app 监听返回按钮

onBackPress(e){
  console.log("监听返回按钮事件",e);
  uni.navigateTo({
    url:"/pages/details/details?type=2"
  })
  // 此处一定姚要return为true，否则页面不会返回到指定路径
  return true;
}
当用户进行以下操作时，会触发该函数：
Android 实体返回键 (from = backbutton)
顶部导航栏左边的返回按钮 (from = backbutton)
返回 API，即 uni.navigateBack() (from = navigateBack)
只有在该函数中返回值为 true 时，才表示不执行默认的返回，自行处理此时的业务逻辑。
当不阻止页面返回却直接调用页面路由相关接口（如：uni.switchTab）时，可能会导致页面显示异常，可以通过延迟调用路由相关接口解决。
不返回或返回其它值，均会执行默认的返回行为。
H5 平台，顶部导航栏返回按钮支持 onBackPress()，浏览器默认返回按键及Android手机实体返回键不支持 onBackPress()
